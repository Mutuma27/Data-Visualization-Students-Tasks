User Interaction Design Documentation

Dashboard Architecture

Built as a single-page web dashboard using Chart.js 4.4.1 for all visualisations. The system is fully client-side with no backend dependency, enabling offline use and rapid deployment.

All 481 survey records are embedded inline. Filter operations run in real time using JavaScript array methods, recomputing all derived statistics on each interaction.

Interactive Controls

  • Gender slicer isolates Male or Female subsets across all charts and KPIs simultaneously.
  • Occupation slicer segments by University Student, School Student, Salaried Worker, or Retired.
  • Relationship slicer filters by Single, In a relationship, Married, or Divorced.
  • Metric toggles add or remove any of 6 mental health indicators from the trend line chart.
  • Reset control restores all filters to the full 481-respondent baseline in one click.

KPI Tracking and Performance Metrics

  • Total respondents in current filter scope
  • Heavy usage rate (more than 5h/day) as a percentage
  • Purposeless usage rate (scores 4 to 5 on Likert scale)
  • Average depression score across usage brackets
  • Average sleep disruption score across usage brackets
  • Most adopted platform in current filter scope

All KPIs recalculate live on every filter change.

User-Centred Design Principles

  • Every chart and KPI card carries a contextual tooltip (i icon) explaining what the metric means, how to read it, and what the data shows.
  • Colour coding is consistent: blue for neutral data, red for risk indicators, green for adoption metrics.
  • Screen-reader accessible via aria-label attributes on all canvas elements and an sr-only heading.
  • Responsive grid layouts reflow at 960px, 660px, and 540px breakpoints for tablet and mobile support.
  • Hover states on every interactive element provide immediate affordance feedback.

Filter: Gender = Female, then Male

Model: Logistic Regression · L2 penalty · C=1.0 · StandardScaler normalisation · 10-fold CV mean accuracy 98.5%
▶ Model Technical Details & Methodology (M6 Documentation) Logistic Regression · L2 · AUC 0.999

Pipeline Architecture

Step 1 StandardScaler: Each of the 13 features is normalised to zero mean and unit variance using parameters fitted only on the training set (382 respondents), preventing data leakage into evaluation.

Step 2 LogisticRegression: L2 (Ridge) penalty with C=1.0 shrinks coefficients to reduce overfitting. Solver: lbfgs. Max iterations: 1000.

Feature Selection Rationale

Only the 12 raw Likert-scale survey items (Q9–Q20) plus Daily Minutes (Q8) are used as predictors. Composite scores (Addiction Score, MH Impact Score, etc.) are deliberately excluded to avoid data leakage the binary target is derived from those same composites.

Target Encoding

The four-tier MH Risk label is collapsed to binary: High Risk + Very High Risk → 1, Low Risk + Moderate Risk → 0. This corresponds to a MH Impact Score above 3.0/5 and represents the clinically meaningful intervention threshold.

Class balance: ~55% High Risk, ~45% Low/Moderate Risk approximately balanced, no oversampling required.

Validation Strategy

Hold-out test set: 20% stratified split (96 respondents). Stratification preserves the original class ratio in both train and test splits.

10-fold stratified cross-validation on the full dataset confirms stability: mean accuracy 98.5% (±1.2%), mean AUC 0.999 (±0.001).

Dashboard Integration Method

Model parameters (13 coefficients, intercept, scaler means/stds) are extracted from the trained Python pipeline and hardcoded as JavaScript constants. The browser replicates the exact same sigmoid calculation: P = 1 / (1 + exp(−z)) where z = intercept + Σ(coefᵢ × (xᵢ − μᵢ) / σᵢ).

Optimal classification threshold: 0.647 (Youden index from ROC curve).

Key Findings

Top predictors of High Risk classification (by coefficient magnitude): Depression Frequency (+1.78), Validation Seeking (+1.68), Distraction While Busy (+1.54), Social Comparison (+1.47). Daily Minutes has the weakest effect (+0.38), replicating the M4 regression finding that behavioural quality matters more than raw screen time.
